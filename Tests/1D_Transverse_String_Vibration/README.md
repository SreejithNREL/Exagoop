# Test Case: 1D Transverse String Vibration

## Physical Problem

A one-dimensional elastic rod is used to model transverse (out-of-plane) vibration analogous to a vibrating string. The material occupies the interval $x \in [0, L]$ with $L = 1$ m, Young's modulus $E = 50$, density $\rho = 25$, and zero Poisson ratio. The string is initialised with a sinusoidal transverse velocity corresponding to the fundamental mode and subsequently oscillates freely. The transverse displacement is captured in the $y$ coordinate of the material points. This test is primarily a qualitative demonstration of transverse wave propagation in MPM; the main output is an animation of the vibrating profile.

## Exact Solution

The transverse wave equation for small amplitudes is:

$$\frac{\partial^2 v}{\partial t^2} = c^2 \frac{\partial^2 v}{\partial x^2}, \quad c = \sqrt{E/\rho} = \sqrt{50/25} = \sqrt{2} \approx 1.414 \text{ m/s}$$

With periodic boundary conditions in the transverse directions and the initial sinusoidal velocity $v(x,0) = V_0 \sin(n\pi x / L)$ ($n = 1$, $V_0 = 0.1$), the exact transverse velocity evolves as:

$$v(x, t) = V_0 \sin\!\left(\frac{\pi x}{L}\right) \cos(\omega_1 t), \quad \omega_1 = \frac{\pi c}{L}$$

No automated validation script is included for this test; comparison with the analytical solution can be performed by adapting `PostProcess/plot_animation.py`.

## Simulation Setup in ExaGOOP

The setup is generated by `PreProcess/generate_particle_and_inputfiles.py`. Default parameters:

| Parameter | Value |
|-----------|-------|
| Dimensions | Nominally 3D (1D string with buffer cells in $y$, $z$) |
| Domain length | $L = 1$ m, 25 cells in $x$ |
| Buffer cells (y, z) | 3 each side |
| Particles per cell (x) | 1 |
| Young's modulus | $E = 50$ |
| Density | $\rho = 25$ |
| Wave speed | $c = \sqrt{2} \approx 1.414$ m/s |
| Order scheme | 1 |
| PIC/FLIP | Pure FLIP ($\alpha = 1.0$) |
| Stress update | MUSL |
| CFL | 0.1 |
| Periodicity | Periodic in $y$ and $z$ |
| Gravity | Zero |
| Initial velocity | Sinusoidal $V_0 \sin(\pi x / L)$ in $y$-direction |

The buffer cells in $y$ and $z$ around the string prevent the material points from interacting with hard boundaries in the transverse directions. Periodicity ensures waves travel unimpeded.

## Key Input Parameters

All parameters are passed as command-line arguments to `PreProcess/generate_particle_and_inputfiles.py`:

```
python3 PreProcess/generate_particle_and_inputfiles.py \
    <no_of_cells_in_x> <buffery> <periodic> <np_per_cell_x> \
    <order_scheme> <alpha_pic_flip> <stress_update_scheme> <CFL> <output_tag>
```

The key parameters and their roles are:

- **`no_of_cells_in_x`** — grid resolution in the string direction. Finer grids resolve higher modes.
- **`buffery`** — number of buffer cells in $y$ (and $z$). Must be at least 1; 3 is typical.
- **`np_per_cell_x`** — particles per cell along the string. Increasing this improves spatial accuracy.
- **`alpha_pic_flip`** — `1.0` for pure FLIP (no dissipation); reduce towards 0 to damp high-frequency oscillations.
- **`CFL`** — time step fraction. Must satisfy $\Delta t \le \text{CFL} \cdot h / c$.

## Post-Processing

### Prerequisites

```bash
pip install numpy matplotlib
```

### Step 1 – Generate particles and input file

```bash
python3 PreProcess/generate_particle_and_inputfiles.py \
    25 3 1 1 1 1.0 1 0.1 my_string_run
```

This writes `mpm_particles.dat` and the `.inp` file.

### Step 2 – Run ExaGOOP

```bash
cp /path/to/ExaGOOP3d.*.ex .
mpirun -n 4 ./ExaGOOP3d.*.ex inputs_axialbar.in
```

Output particle files are written to `Solution/particle_files/`.

### Step 3 – Animate the transverse displacement

```bash
python3 PostProcess/plot_animation.py
```

Reads all `matpnt_t*` files from `Solution/ascii_files/` and produces an animated scatter plot of particle positions (x vs y), showing the standing-wave oscillation. A second variant is available as `plot_animation1.py`.

To export the animation to a file, modify the `plt.show()` call to use `ani.save('string_vibration.gif', writer='pillow')`.
