# Test Case: 2D Dam Break with Obstacles (Embedded Boundary)

## Physical Problem

This is an extension of the `Dam_Break` test case. The same $0.1 \times 0.2$ m water column collapses in a $0.4 \times 0.4$ m tank, but a solid rectangular obstacle is placed in the path of the flow. The obstacle is represented using the **Embedded Boundary (EB)** level-set framework, allowing arbitrarily shaped solid obstacles without body-fitted meshing. Two obstacle configurations are provided:

- **Box obstacle** (`config_box_obstacle.json`): a rectangular block at $x \in [0.15, 0.25]$, $y \in [0, 0.1]$ m.
- **STL obstacle** (`config_stl_obstacle.json`): an obstacle defined from an STL surface mesh, loaded via the level-set framework.

The default `config.json` uses a circular (sphere-type) level-set body centred at $(0.2, 0.07)$ with radius 0.035 m. This test exercises free-surface fluid flow over and around embedded solid obstacles.

## Exact Solution

No closed-form exact solution exists. Validation is qualitative and by comparison with experimental waterfront data (same Martin & Moyce dataset as `Dam_Break`). The presence of the obstacle deflects and delays the water front; the waterfront position vs time differs from the obstacle-free case. Key features to examine: wave reflection off the obstacle, run-up on the obstacle face, and the post-impact flow pattern.

### Box obstacle configuration (`config_box_obstacle.json`)

The obstacle is a rectangular block defined by `box_lo: [0.15, 0.0]` and `box_hi: [0.25, 0.1]`. The `geom_type: "box"` EB shape is used, with EB enabled (`use_eb: true`).

### STL obstacle configuration

An STL-based obstacle is loaded from an STL file generated by `PreProcess/generate_box_stl.py`. Run this script first to produce the STL file.

## Key Input Parameters

- **`order_scheme: 3`** — cubic basis functions (vs order 1 in the plain Dam_Break). Higher-order schemes reduce numerical diffusion at the free surface but require more particles per cell for stability.
- **`use_eb: true`** — enables the embedded boundary solver. Required for the level-set obstacle.
- **`levelset_bodies[0].geom_type`** — `"sphere"` (circular), `"box"` (rectangle), or `"stl"` (STL file). Controls the obstacle shape.
- **`levelset_bodies[0].sphere_center`** / **`box_lo`** / **`box_hi`** — obstacle position and size.
- **`levelset_bodies[0].ls_refinement`** — level of EB grid refinement for interface accuracy.
- **`levelset_bodies[0].levelset_mom`** — `"slipwall"` (frictionless obstacle surface). Could be changed to `"noslip"` for viscous wall effects.
- **`diagnostics.do_calculate_minmaxpos`** — tracks the waterfront position for comparison with experiments.
- **`output_format: "ascii"`** — ASCII output (vs HDF5 in plain Dam_Break), which is compatible with the gnumake build without HDF5.

## Generating the STL Obstacle

```bash
python3 PreProcess/generate_box_stl.py
```

This writes a simple box STL file that can be used with `config_stl_obstacle.json`.

## Post-Processing

### Prerequisites

```bash
pip install numpy matplotlib
```

### Step 1 – Generate particles and input file

Choose the desired configuration and run:

```bash
# For the default (sphere obstacle):
python3 PreProcess/Generate_MPs_Inputfile_Generic.py --config PreProcess/config.json

# For the box obstacle:
python3 PreProcess/Generate_MPs_Inputfile_Generic.py --config PreProcess/config_box_obstacle.json
```

### Step 2 – Build and run ExaGOOP

An EB-enabled build is required:

```bash
# gnumake (EB-enabled):
cd /path/to/ExaGOOP_Dev
make -f GNUmakefile USE_EB=TRUE USE_MPI=TRUE DIM=2
cp ExaGOOP2d.*.ex Tests/Dam_Break_With_Obstacles/
cd Tests/Dam_Break_With_Obstacles
mpirun -n 4 ./ExaGOOP2d.*.ex Inputs_DamBreak_Obstacles.inp
```

### Step 3 – Plot waterfront and animation

```bash
python3 PostProcess/plot_waterfront.py \
    --folder     Solution/ascii_files/<output_tag> \
    --minmaxfile Solution/diag_<output_tag>.dat \
    --expdata    /path/to/martin_moyce_1952.dat \
    --outputpic  waterfront_obstacle.png \
    --outputmovie dambreak_obstacle.mp4
```

The animation shows the water column collapsing and impacting the obstacle, with particles coloured by speed. The waterfront plot shows the effect of the obstacle: the front initially follows the obstacle-free trajectory then decelerates upon impact.
